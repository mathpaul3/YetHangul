# Spec v1 Status

이 문서는 현재 **active spec status source of truth**다.
프로젝트의 구현 상태를 요구사항/작업과 분리해서 관리한다.

상태 표기:

- `Done`
- `Partial`
- `Not Started`

## Sections

- 제품 목표
- 플랫폼 정책
- 데이터 표현
- 핵심 구조
- 입력 규칙
- UI / 편집기
- 배포 / 운영

## Rules

1. spec은 제품이 어떤 상태여야 하는지를 적는다.
2. 구현 task나 세부 일정은 `docs/tasks/active.md`에서 관리한다.
3. 완료된 iteration의 spec snapshot은 `docs/specs/archive/`로 옮긴다.
